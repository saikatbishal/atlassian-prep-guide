Advanced React Application Assignment: E-Commerce Dashboard

Project Overview

You will build a full-featured E-Commerce Dashboard using the DummyJSON API. This project will test your skills in authentication, state management, routing, and data handling.

Core Requirements

1. Authentication System

  • Implement a complete auth flow using DummyJSON's auth endpoints
  • Login page with form validation
  • Store access token and refresh token properly
  • Implement token refresh logic when the access token expires
  • Protected routes that redirect to login if not authenticated
  • Logout functionality that clears auth state
  • Persist authentication state across page refreshes

2. Routing Structure

Create the following routes using React Router:

  • /login - Public route
  • / - Protected: Dashboard home
  • /products - Protected: Products list with pagination
  • /products/:id - Protected: Single product detail
  • /cart - Protected: Shopping cart view
  • /users - Protected: Users list with pagination
  • /users/:id - Protected: User profile view

3. Redux State Management

Set up Redux (or Redux Toolkit) to manage:

  • Auth slice: User data, tokens, authentication status
  • Products slice: Product list, current product, pagination state, filters
  • Cart slice: Cart items, total calculations
  • Users slice: Users list, current user, pagination state

4. Product Management Features

  • Display paginated products list (use limit and skip parameters)
  • Implement product search functionality
  • Filter products by category (fetch and use categories from API)
  • Sort products by price or title
  • View individual product details
  • Add products to cart (cart state managed in Redux)

5. Cart Functionality

  • Add/remove products from cart
  • Update product quantities
  • Calculate cart totals
  • Display cart count in navigation
  • Fetch and sync with user's cart from API after login

6. User Management

  • Display paginated users list
  • Search users by name
  • View individual user profiles
  • Display the authenticated user's information

7. Pagination

Implement reusable pagination for:

  • Products list (30 items per page)
  • Users list (20 items per page)
  • Handle page changes and update URL query parameters
  • Show current page, total pages, and navigation controls

Technical Specifications

API Integration Requirements

Create an API service layer with interceptors for:

  • Adding auth tokens to requests
  • Handling token refresh on 401 errors
  • Error handling and logging
  • Use async thunks (Redux Toolkit) or similar patterns for API calls
  • Handle loading states for all async operations
  • Implement proper error handling and display error messages

Advanced Features

  1. Token Refresh Logic: Automatically refresh expired tokens without user intervention
  2. Optimistic Updates: Update Redux state immediately for cart operations, then sync with API
  3. Search Debouncing: Debounce search input to avoid excessive API calls
  4. Route Guards: Create a higher-order component or custom hook for route protection
  5. Persistent Cart: Sync local cart state with the API cart after authentication

State Architecture

  • Use Redux Toolkit with slices
  • Implement selectors for derived state (e.g., cart totals, filtered products)
  • Use proper action creators and reducers
  • Implement middleware for side effects if needed

Evaluation Criteria

You will be evaluated on:

  1. Authentication Flow (25%): Proper token management, refresh logic, and route protection
  2. State Management (25%): Clean Redux architecture, proper data flow, and state updates
  3. API Integration (20%): Efficient API calls, error handling, and token refresh interceptors
  4. Pagination (15%): Smooth pagination experience across different views
  5. Code Organization (15%): Component structure, reusability, and separation of concerns

Bonus Challenges (Optional)

  • Implement product filtering with multiple simultaneous filters (category + price range)
  • Add infinite scroll as an alternative to traditional pagination for products
  • Create a "recently viewed products" feature using local storage
  • Implement a search history feature
  • Add loading skeletons for better UX during data fetching

DummyJSON Endpoints You'll Need

  • POST /auth/login - Authentication
  • POST /auth/refresh - Token refresh
  • GET /auth/me - Current user
  • GET /products - Products list
  • GET /products/search - Search products
  • GET /products/categories - All categories
  • GET /products/category/{category} - Products by category
  • GET /products/{id} - Single product
  • GET /carts/user/{id} - User's cart
  • GET /users - Users list
  • GET /users/search - Search users
  • GET /users/{id} - Single user

Submission Guidelines

Your application should:

  • Run without errors
  • Handle edge cases (empty states, API errors, expired tokens)
  • Maintain authentication state across page refreshes
  • Have a logical routing structure
  • Use Redux for all shared state
  • Implement proper loading and error states

Good luck! This project will demonstrate your ability to build production-ready React applications with complex state management and authentication flows.
